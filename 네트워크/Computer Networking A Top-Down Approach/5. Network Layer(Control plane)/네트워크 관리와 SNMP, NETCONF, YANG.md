# 네트워크 관리의 주요 구성 요소
- **관리 서버**: 네트워크 운영 센터(NOC)에서 에이전트가 설치된 장치들을 모니터링하고 제어한다. 이들은 관리 프로토콜을 통해 통신한다.
- 관리 대상 디바이스: 관리되는 네트워크에 있는 장비(소프트웨어 포함)다. 호스트, 라우터, 스위치, 미들박스, 모뎀, 온도계 등일 수 있다.
- **데이터**: 관리 대상 디바이스에 있는 여러 유형의 데이터들로, 네트워크 관리자는 원격으로 디바스 데이터를 조회하거나 수정하여 제어할 수 있다. 관리 서버는 네트워크 전반의 데이터(e.g. 네트워크 토폴로지)도 자체적으로 유지 관리한다.
- **네트워크 관리 에이전트**: 관리 대상 디바이스에서 실행되는 소프트웨어 프로세스로, 관리 서버의 명령과 제어 하에 해당 디바이스에서 작업을 수행한다. 라우팅 에이전트와 유사하다.
- **네트워크 관리 프로토콜**: 관리 서버와 관리 대상 디바이스 간에 사용되어 네트워크 관리 에이전트가 작업을 수행하고 관리 서버에 이상 상황을 알릴 수 있게 한다. 즉, 네트워크 관리 프로토콜 자체가 네트워크를 관리하는 것이 아니라, 네트워크 관리자가 네트워크를 관리하는데 사용할 수 있는 기능을 제공하는 것이다.
## 네트워크 관리자가 구성 요소를 사용하는 방법
- **CLI**
- **SNMP/MIB** (Simple Network Management Protocol/Management Information Base): 네트워크 관리자는 SNMP를 사용하여 디바이스의 MIB 객체에 포함된 데이터를 조회하거나 수정할 수 있다. 디바이스를 개별적으로 관리한다.
- **NETCONF/YANG**: 추상적이고 전체적인 네트워크 관리 관점으로, 정확성 제약 조건 지정 및 여러 관리 대상 디바이스에 대한 원자적 관리 작업 제공을 포함하여 구성 관리에 중점을 둔다. YANG은 구성 및 운영 데이터를 모델링하는데 사용되는 데이터 모델링 언어다. NETCONF 프로토콜은 YANG 호환 작업 및 데이터를 디바이스와 주고 받거나 디바이스 간에 교환하는데 사용된다.
# SNMP & MIB
- SNMPv3은 관리 서버와 에이전트 간에 네트워크 관리 메시지를 전달하는 애플리케이션 계층 프로토콜이다.
- 주로 요청-응답 모드(Get/Set)나 예외 상황을 알리는 트랩(Trap) 메시지 모드로 동작한다.
- MIB는 SMI라는 데이터 기술 언어를 사용하여 관리 대상 디바이스의 데이터 객체(e.g. 카운터, 상태 정보)를 정의한다. MIB 객체들은 관련 기능별로 MIB 모듈(예: IP용 RFC 4293, TCP용 RFC 4022)로 그룹화된다. SNMP PDU는 일반적으로 비신뢰성 UDP를 통해 전송되며, SNMPv3는 이전 버전에 비해 보안 및 관리 기능이 크게 강화되었다.
