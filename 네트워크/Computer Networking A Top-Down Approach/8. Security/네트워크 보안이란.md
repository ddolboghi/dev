# 네트워크 보안의 속성
- **기밀성**: 오직 송신자와 수신자만 메시지의 내용을 이해할 수 있어야 한다. 메시지는 암호화되어야 한다.
- **메시지 무결성**: 전송 중에 메시지는 변경되지 않아야 한다. 이를 위해 체크섬 기술의 확장된 형태가 사용된다.
- **엔드포인트 인증**: 송신자와 수신자 모두 상대방의 신원을 확인할 수 있어야 한다.
- **운영 보안**: 인터넷에 연결되있는 네트워크는 방화벽이나 침입 탐지 시스템(IDS) 같은 장치를 사용하여 공격으로부터 보호한다.
# 침입 행위
- **도청**: 채널상의 제어 및 메시지를 스니핑(sniffing)하고 기록한다.
- **메시지 수정, 삽입, 삭제**: 침입자는 감청하거나, 다른 개체로 위장하거나, 진행 중인 세션을 가로채거나, DoS(서비스 거부) 공격 등을 할 수 있다.
# 보안 계층
- application 계층: 특정 애플리케이션 프로토콜에 보안을 적용하면, 해당 애플리케이션은 기밀성, 인증, 무결성 등의 보안 서비스를 누릴 수 있다.
- transport 계층: transport 프로토콜에 보안을 제공하면, 해당 프로토콜을 사용하는 모든 애플리케이션이 보안 서비스를 받는다.
- network 계층: 호스트 간 보안을 제공하면, 모든 transport 세그먼트와 애플리케이션 데이터가 보호된다.
- link 계층: 링크 기반 보안을 제공하면, 해당 링크를 통과하는 모든 프레임이 보호된다.
