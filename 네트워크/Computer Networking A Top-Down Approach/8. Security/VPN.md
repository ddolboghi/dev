# VPN (Virtual Private Network)의 특징
- 기관은 공용 인터넷과 물리적으로 완전히 분리된 네트워크(사설망, private network)를 사용할 수 있지만, 비용이 많이 든다.
- VPN을 사용하면 기관의 사무실 간 트래픽은 물리적으로 독립적인 네트워크가 아닌 공용 인터넷을 통해 전송된다. 이때 기밀성을 위해 트래픽이 공용 인터넷으로 들어가기 전에 암호화된다.
# VPN의 작동 방식
1. 호스트가 일반 IP 데이터그램을 VPN을 사용하는 다른 호스트에게 보낸다.
2. 호스트 측 게이트웨이 라우터는 IPv4 데이터그램을 IPsec 데이터그램을 변환한 다음 공용 인터넷으로 전달한다.
	- IPsec 데이터그램에는 **IPsec 헤더**와 암호화된 페이로드가 있다.
3. 공용 인터넷의 라우터들은 이 IPsec 데이터그램의 IPv4 헤더를 보고 일반적인 IPv4 데이터그램처럼 처리한다.
4. IPsec 데이터그램이 목적지 호스트에 도착하면 호스트의 OS는 페이로드를 복호화하고 상위 계층 프로토콜(e.g. TCP, UDP)로 전달한다.
