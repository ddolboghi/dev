#  컴퓨터 구조
## 컴퓨터가 이해하는 정보
### 데이터
- 숫자, 문자, 이미지, 동영상 같은 정적인 정보
### 명령어
- 데이터를 움직이고 컴퓨터를 작동시키는 정보

## 컴퓨터의 4가지 핵심 부품
### CPU(central processing unit)
- 메모리에 저장된 명령어를 읽어 들이고, 해석하고, 실행함
- CPU내부에는 ALU, 레지스터, 제어장치가 있음
#### ALU(arithmetic logic unit; 산술논리연산장치)
- 컴퓨터 내부에서 대부분의 계산 수행
#### 레지스터(register)
- CPU내부의 작은 임시 저장 장치
- 프로그램 실행에 필요한 값들을 임시로 저장
- 여러 개의 레지스터가 각기 다른 이름과 역할을 가짐
#### 제어장치(CU; control Unit)
- 제어 버스를 통해 전기 신호를 보내고 명령어를 해석하는 장치
	- CPU가 메모리에 저장된 값을 읽으려할땐 메모리에 메모리 읽기라는 제어신호를 보냄
	- CPU가 메모리에 어떤 값을 저장할땐 메모리에 메모리 쓰기라는 제어신호를 보냄
### 주기억장치(메모리, main memory)
- RAM(random access memory)
- ROM(read only memory)
- 보통 주기억장치는 RAM을 지칭함
- 현재 실행되는 프로그램의 명령어와 데이터를 저장
- 프로그램이 실행되려면 반드시 메모리에 저장되어 있어야함
- 메모리에 저장된 값의 위치는 **주소**로 알 수 있음 
- 메모리의 대역폭이 클수록 데이터를 더 빨리 저장할 수 있음음

### 보조기억장치(secondary storage)
- 전원이 꺼져도 저장된 내용을 잃지 않는 장치
- 메모리가 현재 '실행되는' 프로그램을 저장한다면, 보조기억장치는 '보관할' 프로그램을 저장함
### 입출력 장치(input/output(I/O) device)
- 컴퓨터 외부에 연결되어 컴퓨터 내부와 정보를 교환할 수 있는 부품
- 보조기억장치와 입출력장치를 통틀어서 '주변장치'라 부르기도 함
- 보조기억장치는 입출력장치와 달리 메모리를 보조한다는 특징이 있어서 구분함
## 시스템 버스
- 버스(bus) : 메인보드에 연결된 부품들이 서로 정보를 주고받는 통로
- 시스템 버스 : CPU, 메모리, 주변장치 간 데이터를 주고받는 통로
- 시스템 버스는제어버스, 주소 버스, 데이터 버스로 구성됨
- 버스의 대역폭: 한번에 전달할 수 있는데이터의 최대 크기로, CPU가 한번에 처리할 수 있는 데이터의 크기와 같음
- 버스의 대역폭 == 레지스터의 크기 == 메모리에 한번에 저장할 수 있는 데이터의 크기 = 단위: word
#### 제어 버스
- 제어 신호를 주고받는 통로
- CPU로 제어 신호가 전달됨
- 제어 버스의 신호는 CPU, 메모리, 주변장치와 양방향으로 오고감
#### 주소 버스
- 주소를 주고받는 통로
- CPU의 메모리 주소 레지스터는 주소 버스로 신호를 보내고, 메모리와 주변장치는 신호를 받는 단방향
#### 데이터 버스
- 명령어와 데이터를 주고받는 통로
- CPU의 메모리 버퍼 레지스터와 연결되있고 양방향
# 데이터의 단위
- n bit 는 $2^n$ 가지 정보를 표현할 수 있음
- 워드(word) : CPU가 한번에 처리할 수 있는 데이터 크기. ex) 32bit CPU, 64bit CPU의 bit 크기
- 메모리 주소는 byte 단위
- 메모리에서 데이터를 읽거나 쓸 때는 word 단위

| 단위   |         |
| ------ | ---------- |
| 1 byte | 8 bit      |
| 1 kB   | 1,000 byte |
| 1 MB   | 1,000 kB   |
| 1 GB   | 1,000 MB   |
| 1 TB   | 1,000 GB   |
# 이진수의 음수 표현법
- 이진수의 음수는 **2의 보수**로 표현함
- 2의 보수: 어떤 수를 그보다 큰 $2^n$에서 뺀 값 == **모든 0과 1을 뒤집고, 거기에 1을 더한 값**
# 십육진법
- 수가 15를 넘어가는 시점에 자리 올림
- 코드상에서 십육진수 표기할때는 앞에 `0x`붙임

| 십진수 | 0 | 1 | 9 | 10 | 11 | 12 | 13 | 14 | 15 | 16 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 십육진수 | 0 | 1 | 9 | A | B | C | D | E | F | 10 |
십육진수 A ~ F 쉽게 암기하기
- 손가락을 하나도 안피면 10, A
- 손가락을 하나 피면 11, B
- 손가락을 다 피면 15, F
# 이진수 <-> 십육진수
- 십육진수 숫자 하나를 이진수로 표현하려면 4비트($2^4$)가 필요함
- 십육진수 --> 이진수: 십육진수 숫자 하나가 의미하는 십진수를 4자리(4비트)의 이진수로 표현
- 이진수 --> 십육진수: 이진수 4자리가 의미하는 십진수를 십육진수로 표현
# 명령어의 구조
- 명령어는 연산 코드(연산자)와 operand(피연산자)로 구성됨
- n-주소 명령어: operand가 n개 있는 명령어(0 <= n)
- 연산 코드의 종류는 매우 많음
## 대부분의 CPU가 공통으로 이해하는 기본적인 연산 코드 유형
데이터 전송
- MOVE: 데이터를 옮겨라
- STORE: 메모리에 저장하라
- LOAD(FETCH): 메모리에서 CPU로 데이터를 가져와라
- PUSH: 스택에 데이터를 저장하라
- POP: 스택의 최상단 데이터를 가져와라

산술/논리 연산
- ADD / SUBTRACT / MULTIPLY / DIVIDE: 덧셈 / 뺄셈 / 곱셈 / 나눗셈을 수행하라
- INCREMENT / DECREMENT: operand에 1을 더하라 / operand에 1을 빼라
- AND / OR / NOT 연산을 수행하라
- COMPARE: 두 개의 숫자 또는 TRUE / FALSE 값을 비교하라

제어 흐름 변경
- JUMP: 특정 주소로 실행 순서를 옮겨라
- CONDITIONAL JUMP: 조건에 부합할때 특정 주소로 실행 순서를 옮겨라
- HATL: 프로그램의 실행을 뭠춰라
- CALL: 되돌아올 주소를 저장한 채 특정 주소로 실행 순서를 옮겨라(함수 호출)
- RETURN: CALL을 호출할 때 저장했던 주소로 돌아가라(함수 반환)

입출력 제어
- READ(INPUT): 특정 입출력 장치로부터 데이터를 읽어라
- WRITE(OUTPUT): 특정 입출력 장치로 데이터를 써라
- START IO: 입출력 장치를 시작하라
- TEST IO : 입출력 장치의 상태를 확인하라
## 주소 지정 방식
- operand에 데이터를 넣으면 저장 공간의 한계 때문에 표현할 수 있는 정보의 가짓수가 줄어듬
- operand에 데이터가 있는 메모리 주소인 **유효 주소**를 넣음

| 대표적인 주소 지정 방식 |  |
| ---- | ---- |
| 즉시 주소 지정 방식 | - operand 필드에 연산에 사용할 데이터를 직접 넣음<br>- 데이터를 찾는 과정이 없어 가장 빠름 |
| 직접 주소 지정 방식 | - operand 필드에 유효주소 넣음<br>- 표현할 수 있는 operand 필드의 길이가 연산 코드의 길이만큼 짧아져 유효 주소에 제한이 생길 수 있음 |
| 간접 주소 지정 방식 | - 유효 주소를 메모리에 저장하고 유효 주소의 주소를 operand 필드에 넣음<br>- 직접 주소 지정 방식보다 표현할 수 있는 유효 주소의 범위가 넓지만 두 번의 메모리 접근이 필요해 느림 |
| 레지스터 주소 지정 방식 | - 연산에 사용할 데이터를 저장한 레지스터를 operand 필드에 명시<br>- 표현할 수 있는 레지스터 크기에 제한이 생길 수 있음 |
| 레지스터 간접 주소 지정 방식 | - 유효 주소를 저장한 레지스터를 operand 필드에 명시<br>- 메모리에 한 번 접근해 간접 주소 지정 방식보다 빠름 |
| 스택 주소 지정 방식 | - 메모리에서 스택 영역의 주소를 저장한 스택 포인터를 operand 필드에 명시 |
| 변위 주소 지정 방식 | - operand 필드의 값과 특정 레지스터의 값을 더하여 유효 주소를 얻어냄<br>- 명령어의 구조는 연산 코드 필드, 레지스터 필드, operand 필드임<br>- 어떤 레지스터를 더하는지에 따라 상대 주소 지정 방식, 베이스 레지스터 주소 지정 방식 등이 있음 |
### 상대 주소 지정 방식(relative addressing mode)
- 유효 주소 = 프로그램 카운터의 값 + operand 
- if문과 유사하게 분기하여 특정 주소의 코드를 실행할때 사용
### 베이스 레지스터 주소 지정 방식(base-register addressing mode)
- 유효 주소 = 베이스 레지스터의 값 + operand
- 베이스 레지스터는 기준 주소, operand는 기준 주소로부터 떨어진 거리
